一、Cookie和localStorage、sessionStorage区别？

  1. Cookie：最初用在客户端存储回话信息，后来要求服务器在响应http请求时，通过发送set-cookieHTTP包含头部会话信息，简单来说（cookie是服务器端发给客户端的文本文件；目的是用于辨别用户身份）。构成包含name、value、domain(域名)、path、secure安全标准(选中为https，默认http无状态协议)、(expire)过期时间;还包含sessionStorage:回话信息，

*cookie服务器通过set-cookie给前段application发送的信息

    二、什么是回流，什么是重绘，有什么区别？

    答：当render tree中的一部分(或全部)因为元素的规模尺寸，布局，隐藏等改变而需要重新构建。这就称为回流(reflow)。
    当render tree中的一些元素需要更新属性，而这些属性只是影响元素的外观，风格，而不会影响布局的，比如background-color。则就叫称为重绘(repaint)。
    区别：
    回流必将引起重绘，而重绘不一定会引起回流。比如：只有颜色改变的时候就只会发生重绘而不会引起回流

studyAjax

  • XHR基本步骤
  • method方法
    1. get 查询数据，url带着查询参数
    2. post 新增数据,数据放在request body不在URL中
    3. put 更新数据,全部更新
    4. patch(打补丁) 更新数据,局部更新
    5. delete 更新数据 全部更新
    6. head 获取数据的信息,不要数据内容
    7. option 测试服务器支持那些方法

接口规范

  • resetful 风格的接口
    1. get /api/books：获取所有书籍数据
    2. get /api/books/007 获取id为007的一本书
    3. get /api/books/？pagenum=1&pagesize = 10 获取第一页十条数据
    4. post /api/books 增加
    5. post /api/books/007 修改id为007的这本书 全量更新 用数据完全覆盖后台数据，只保留id
    6. patct /api/books/007 修改id为007的这本书 局部更新 用数据修改后台数据，如果有新增， 增加
    7. delete /api/books/007 删除id为007的这本书

辅助模拟服务器

  1. 安装 npm install -g json-server
  2. 启动服务器json-server -w books.json
  3. 延时操作 json-server -w -d 3000 books.json
